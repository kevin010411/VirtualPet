# species_by_healthy_days.txt
# Purpose:
#   Let SD-card data decide which species should be used after the pet has lived
#   a certain number of healthy days. The outfit field is the default outfit for
#   a newly selected species.
#
# Location suggestion on SD card:
#   /species_by_healthy_days.txt
#
# Line format:
#   min_healthy_days|max_healthy_days|species|outfit
#
# Rules:
#   - Lines starting with # are comments.
#   - Empty lines are ignored.
#   - min_healthy_days and max_healthy_days are inclusive.
#   - Use * for max_healthy_days to mean no upper limit.
#   - species and outfit must be 1-8 characters.
#   - species and outfit may use only A-Z, a-z, 0-9, _ and -.
#   - Put more specific ranges before wider fallback ranges.
#   - If no row matches, firmware should keep the current species/outfit.
#
# Evolution behavior:
#   - healthy_days is stored with the pet state in /state_a.bin or
#     /state_b.bin.
#   - healthy_days increases only when dayPassed() runs while the pet status is
#     Healthy. Hungry, Depressed, Dirty, Poop, or Sick days do not increase it.
#   - Firmware checks this file after loading a pet, after resetting a pet, and
#     after healthy_days increases.
#   - The first matching row is applied only when its species differs from the
#     current species. When the species is unchanged, firmware keeps the current
#     outfit so manual outfit changes are not overwritten.
#   - When the species changes, firmware applies that row's outfit as the new
#     species default outfit, reloads the animation manifest, and saves state.
#   - If this file is missing, invalid, or has no matching row, firmware keeps
#     the current species/outfit.
#   - Firmware no longer supports old state.bin/state.bak files. If neither
#     state slot is valid, the pet starts from a fresh default state.
#
# Example:
#   A pet starts as dino/base.
#   After 3 healthy days it becomes baby/base.
#   After 7 healthy days it becomes teen/base.
#   After 14 healthy days it becomes drgn/base.

0|2|dino|base
3|6|baby|base
7|13|teen|base
14|*|drgn|base
